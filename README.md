体验小模型

**验证结论：小模型办大事**
经过对比测试，得出以下核心结论：
1. **必须前置处理**：不能直接把 193 篇笔记直塞给大模型（首轮慢、多轮崩）。
2. **本地向量化可行**：使用本地轻量级模型（如 `BAAI/bge-small-zh-v1.5`）可以将检索精度提升到可用量级。
3. **架构闭环**：本地小模型负责“精准捞针”（毫秒级），云端大模型负责“深度总结”（秒级）。这就像之前做人脸照片向量化检索一样，需要一个“本地测绘员”。

## 阶段进展：小模型和大模型合体 (DocMind v1.0)

目前已完成从“直塞方案”向 **RAG (检索增强生成)** 架构的全面进化。通过本地感知与云端逻辑的合体，彻底解决了 193 篇笔记导致的“大脑载荷爆炸”问题。

### 核心升级点：
1. **意图重写 (Query Rewriting)**：
   - 解决了代词消歧问题。当用户问时，系统会自动结合历史，将其重写再进行检索。
   
2. **精确拦截器 (Interceptor)**：
   - 解决了语义检索的“盲区”。当用户直接点名“.txt”时，系统会跳过向量匹配，直接提取该文件内容。

3. **短期记忆缓冲区 (Memory Buffer)**：
   - 绕过了 SDK 属性限制，本地维护最近的问答历史，确保在“直觉模式”和“重写模式”之间无缝切换。

4. **启动加速 (Caching)**：
   - 引入 `.npz` 向量持久化缓存。

5. **直觉模式**：
   - 对简单的“你好”、寒暄等短句开启极速响应，跳过不必要的意图重写和重度检索，体验更加丝滑。

## 🚀 阶段进展：影子索引与深度理解 (DocMind v2.0)

如果说 v1.0 是给 AI 装上了“翻书的手”，那么 v2.0 就是给 AI 换了一颗**“懂行的脑子”**。不再仅仅依赖关键词的字面匹配，而是通过 **AI 影子索引 (AI Shadow Indexing)** 赋予了系统洞察文字背后逻辑的能力。

### 核心进化结论：

1. **纯向量化是不够的**：随手记的特征是“乱”。字面上的向量匹配会被语义噪声干扰。
2. **AI 影子索引是“破局点”**：在本地向量化之前，先让 AI 进行“透视”，提取出笔记中隐藏的项目归属、情绪标签和技术深度。
3. **架构闭环 2.0**：
* **本地 3060Ti (Ollama)**：作为“保密局特工”，负责全量笔记的影子索引生成，确保隐私不出户。
* **本地 BGE 小模型**：作为“高速导航员”，基于 AI 增强后的标签进行毫米级检索。
* **云端 Gemini Flash**：作为“首席架构师”，负责跨越文档边界，将底层代码细节串联成宏观项目逻辑。

---

### v2.0 关键升级黑科技：

1. **AI 影子索引 (AI Shadow Indexing)**：
* 解决了“名不副实”的问题。通过预处理，为每篇笔记生成“影子标签”，让检索器能搜到全篇没写的。

2. **架构师思维推理 (Architectural Reasoning)**：
* AI 不再仅仅是总结摘要，它学会了**“向上推断”**。它能意识到底层在整个链路中的角色。

3. **隐私隔离预处理 (Privacy-First)**：
* 引入 **Ollama (Qwen2.5)** 方案，将最敏感、最繁重的全量数据扫描流程本地化，解决了云端 503 报错和隐私泄露的双重痛点。

4. **防语义劫持 (Semantic Anti-Hijacking)**：
* 通过提高检索阈值（0.55）和逻辑隔离原则，严防污染。

## 🚀 阶段进展：逻辑死锁破解与抗幻觉 (DocMind v2.1)

在 v2.0 赋予 AI “懂行的脑子”后，系统遭遇了经典的 RAG “粉红大象”问题——注意力死锁与短名劫持。v2.1 引入了多重逻辑锁和降敏机制，让 AI 从“聪明但固执”进化为“克制且清醒”。

### 核心进化结论：

1. **记忆不是越长越好**：意图重写在保留上下文时，极易造成“前任焦点”阴魂不散。必须赋予系统“强制遗忘”的能力，做到收放自如。
2. **宁可说不知道，也绝不胡编乱造**：作为私人记忆外脑，抗幻觉（Anti-Hallucination）是系统的绝对底线，信任一旦崩塌就很难重建。

### v2.1 关键升级黑科技：

1. **精确拦截降敏 (Smart Interceptor)**：

* 放弃粗暴的字符串包含逻辑，引入基于正则表达式的严格单词边界匹配（`\b`），并提高长度阈值，彻底解决了“短名劫持”问题。

2. **动态焦点释放 (Dynamic Focus Release)**：

* 引入临时黑名单机制 (`ignored_file`)。当检测到话题转移意图（如“其他”、“从xxx出发”）时，将前任焦点打入冷宫，物理切断系统的上下文依赖死锁，让大模型不再“钻牛角尖”。

3. **防历史回退污染 (Fallback Sanitization)**：

* 移除了危险的正则兜底逻辑，防止 AI 抓取自身历史输出作为查询词导致的原地打转。在意图重写彻底失败时，系统将诚实地回退至用户原句，不再画蛇添足。

4. **绝对诚实指令 (Anti-Hallucination Directives)**：

* 向云端架构师（Gemini）下达最高级别的“拒绝胡诌”指令。面对知识库外的宏大问题（如“世界技术背景”），系统不再强行缝合无关代码片段，而是守住底线，勇敢承认局限。

### 🚀 DocMind v3.0：全格式破甲与混合检索版 

经过数次硬核迭代，DocMind 现已进化为具备“视觉破甲”和“跨文档侦探推理”能力的工业级个人外脑。

#### ✨ 核心黑科技 (Core Features)

* **🗡️ 混合检索暴击 (Hybrid Search Boost)**：针对传统 RAG 的“向量稀释”痛点，在 BGE 语义检索（Dense）的基础上，引入自研的“关键词暴击”机制（Sparse）。即使是只有几百字的关键证据，只要命中核心词，也能瞬间突破长文本的压制，实现法庭级的精准取证。
* **🛡️ 扫描件金身破拆 (OCR Sidecar)**：原生支持 `.txt`, `.md`, `.docx`, `.pdf` 全格式。针对无法选中文本的“扫描件/纯图转曲” PDF，系统会自动触发 `RapidOCR` 视觉引擎进行“强行破拆”，并生成 `.ocr.txt` 伴生缓存文件。一次破拆，终身秒读。
* **⏳ 时间线与全局视野感知**：向云端大模型注入全局文件树、文件大小及笔记创建的时间跨度。AI 能够跨越年份，梳理你的技术演进轨迹或事件发展脉络。
