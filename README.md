# 🧠 DocMind

一个基于本地文档的智能问答系统，  
支持多格式解析（TXT/MD/DOC/DOCX/PDF/图片 OCR）、向量检索（RAG）与多轮对话。

👉 面向“分散文档与资料难以统一检索与理解”的问题，提供对话式整理与查询能力。

---

## ✨ 项目背景

日常使用中，文档与资料通常以 txt / md / doc / docx / pdf / png / jpg 等形式分散存储：

- 可以查到内容，但不方便统一查找
- 可以找到信息，但不容易整理与理解
- 可以继续思考，但不方便基于已有内容往下追问

DocMind 尝试在这些场景下提供一个更自然的使用方式。

---

## 🚀 可以做什么

### 1. 查找并简单整理内容

可以从多个文档中找到相关内容，但不只是简单返回，还会做一些整理，例如：

- 最近在做什么
- 最近遇到哪些问题

---

### 2. 支持连续提问

可以基于上一轮结果继续提问，例如：

- “刚才提到的问题是哪个模块？”

系统会结合上下文进行回答。

---

### 3. 在已有结果上继续细化

可以在已有结果基础上继续提问，并尽量在原有范围内查找，减少偏离。

---

### 4. 对记录做简单整理

可以把分散的信息整理成更结构化的形式，例如：

- 按时间线梳理
- 汇总关键事件
- 将不同类型记录放在一起查看

---

### 5. 文件级操作（重命名 / 删除）

支持在对话中发起文件改名和删除操作，采用“预览 -> 确认 -> 执行”流程：

- 重命名：先展示变更预览，再执行
- 删除：软删除到 `.docmind_trash/日期/`，不直接物理抹除
- 变更后会同步更新当前会话里的仓库状态

---

### 6. 文件变更历史查询

支持查询近期文件变更历史（重命名 / 软删除）：

- 可按“最近 N 条”查看
- 输出包含 `event_id`、变更前后路径、SHA 摘要与时间
- 变更记录落库到本地 SQLite（`file_change_log.db`）

---
## 🧩 核心能力（工程视角）

- 基于 embedding + 向量检索 + LLM 的 RAG 问答链路
- 多轮对话状态机（支持追问 / 纠偏 / 上下文约束）
- 候选集约束与状态归并（结果集追问可持续继承）
- 增量索引与本地缓存机制，提升响应效率
- 本地小模型参与查询改写与标签提取（降低成本）
- 无主语追问本地短路，避免无效远程调用
- 文件系统操作闭环（重命名 / 删除 / 历史追踪）
- 调试问句追踪与可观测日志（便于排查路由/检索问题）

## ▶️ 运行方式

```bash
pip install -r requirements.txt
python ask_notes.py
```

可选参数与环境变量：

- `--notes-dir <path>`：指定知识库目录（优先级高于环境变量）
- `DOCMIND_NOTES_DIR=<path>`：指定默认知识库目录
- `DOCMIND_MATCH_LOG_LIMIT=<int>`：控制命中明细日志最大行数（默认 60）
- `DOCMIND_DEBUG_SAVE_QUESTIONS=1`：开启问句追踪日志（写入 `logs/debug_questions/`）
- `DOCMIND_DEBUG_SAVE_DIR=<path>`：自定义问句追踪输出目录
- `DOCMIND_SMALLTALK_LOCAL_LLM=0/1`：关闭/开启闲聊场景本地 LLM 回答（默认 1）
- `DOCMIND_OUT_OF_SCOPE_LOCAL_LLM=0/1`：关闭/开启越界问题本地 LLM 回答（默认 1）
- `DOCMIND_SMALLTALK_REWRITE_TIMEOUT=<sec>`：闲聊补判改写超时（0.5~10，默认 2.5）
- `DOCMIND_EMOJI_ENABLED=0/1`：关闭/开启表情回复（默认 1）
- `DOCMIND_EMOJI_LEVEL=<0|1|2>`：调节表情强度
- `DOCMIND_LOG_FILE=<path>`：日志文件路径（默认 `logs/docmind_sys.log`）
- `DOCMIND_LOG_BACKUP_COUNT=<int>`：日志保留份数
- `DOCMIND_LOG_ROTATE_WHEN=<unit>` / `DOCMIND_LOG_ROTATE_INTERVAL=<int>`：日志切分策略
- `DOCMIND_LOG_ROTATE_UTC=0/1`：日志切分是否按 UTC 计算

说明：参数/环境变量名中的 `notes` 为历史兼容命名，实际可指向任意本地文档/资料目录。

👉 示例数据见：examples/demo_notes_public/

## 🧠 一些实现方式

### 对话状态的简单区分

将问题大致分为几类：

- 普通查询
- 基于内容的追问
- 整理类请求（如时间线）
- 基于结果的追问

不同类型采用不同处理方式。

---

### 查询构建的调整

根据问题类型，对检索词做一些处理：

- 初始问题尽量保持稳定
- 多轮提问保留上下文信息
- 整理类请求适当放宽限制
- 基于结果的追问尽量限制范围

---

### 避免无关扩展

在构建查询时尽量避免引入与原问题无关的词，  
同时在需要整理信息时允许一定扩展。

---

### 本地索引

- 使用本地 embedding 模型进行编码
- 支持增量更新
- 使用缓存减少重复计算

## 🛠️ 技术栈

- Python

- 增量索引与缓存机制
- 文件变更记录（SQLite）与可追溯查询
