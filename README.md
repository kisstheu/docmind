# 🧠 DocMind

DocMind 是一个基于本地笔记的问答与整理工具，  
用于把分散的记录整理成更容易查找和理解的内容。

---

## ✨ 项目背景

日常使用中，笔记通常以 txt / md / docx / pdf 等形式分散存储：

- 可以查到内容，但不方便统一查找
- 可以找到信息，但不容易整理与理解
- 可以继续思考，但不方便基于已有内容往下追问

DocMind 尝试在这些场景下提供一个更自然的使用方式。

---

## 🚀 可以做什么

### 1. 查找并简单整理内容

可以从多个笔记中找到相关内容，但不只是简单返回，还会做一些整理，例如：

- 最近在做什么
- 最近遇到哪些问题

---

### 2. 支持连续提问

可以基于上一轮结果继续提问，例如：

- “刚才提到的问题是哪个模块？”

系统会结合上下文进行回答。

---

### 3. 在已有结果上继续细化

可以在已有结果基础上继续提问，并尽量在原有范围内查找，减少偏离。

---

### 4. 对记录做简单整理

可以把分散的信息整理成更结构化的形式，例如：

- 按时间线梳理
- 汇总关键事件
- 将不同类型记录放在一起查看

---

## 🎯 示例能力

一个典型的使用过程如下：

| 提问 | 作用 |
|------|------|
| 最近在做什么 | 查看整体情况 |
| 最近的技术问题 | 汇总多个记录 |

## 🧠 一些实现方式

### 对话状态的简单区分

将问题大致分为几类：

- 普通查询
- 基于内容的追问
- 整理类请求（如时间线）
- 基于结果的追问

不同类型采用不同处理方式。

---

### 查询构建的调整

根据问题类型，对检索词做一些处理：

- 初始问题尽量保持稳定
- 多轮提问保留上下文信息
- 整理类请求适当放宽限制
- 基于结果的追问尽量限制范围

---

### 避免无关扩展

在构建查询时尽量避免引入与原问题无关的词，  
同时在需要整理信息时允许一定扩展。

---

### 本地索引

- 使用本地 embedding 模型进行编码
- 支持增量更新
- 使用缓存减少重复计算

## 🛠️ 技术栈

- Python
- BGE 向量模型（本地部署）
- 基于 embedding 的语义检索
- 多格式文档加载（txt / md / docx / pdf）
- 自实现对话状态管理（多轮问答 / 追问）
- 增量索引与缓存机制

---

## ⚠️ 当前范围

当前示例主要用于验证：

* 多轮对话
* 基本检索与整理
* 简单的结构化输出

* 大规模文档处理
* OCR 场景优化
